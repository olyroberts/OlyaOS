How to use this dashboard

Time tracking — three ways

The dashboard offers three independent ways to track time. Everything that lands in the Log contributes to your billing totals; nothing else does.

  • Task timer — small ▶ Start on each task in the Plan. Use when you're working on one specific task and want time attributed to it. When you tick a task done (or it gets archived on Export), the time moves to the Log as a "task" entry.
  • Session timer — the big timer at the top. Use when you sit down to work, regardless of which task. Press Start → work → Pause if you need to step away → Stop & Save when done. The dialog lets you describe what you did, optionally link to a task. Keeps running even if you close the browser tab (resumes from the same point next time you open the file).
  • Manual entry — "+ Add manual log entry" button next to the session timer. Use when you forgot to start a timer or worked offline. Set a date in the past, enter minutes worked, write what you did. Goes straight to the Log.

Important: "Total tracked" only counts time that is in the Log. Time currently being accumulated by a running session or task timer does NOT count until it's saved to the Log. This makes billing totals trustworthy — only completed, recorded work counts.

Plan tab

  • Tasks are organised into phases. Click any phase title or task text to edit it.
  • Use ↑ and ↓ arrows on a phase header to reorder phases.
  • Notes button toggles a notes field on the task — for thoughts, questions, ideas related to that specific task.
  • Tick the checkbox when done. Completed tasks stay visible (struck through) until your next Export — so you can see what you accomplished this session. After Export they move to the Log permanently.
  • Add new tasks with the input row inside each phase. Add new phases at the bottom.

Log tab

  • All recorded work lives here. Each entry is tagged: Task, Session, or Manual.
  • Search by text or filter by type and phase.
  • Log entries are not editable — this is your source of truth for time worked. You can Delete an entry if it was a mistake, and Restore a "task" entry to put it back into the Plan as an open task.

Calendar tab

  • Visual overview of work activity by day. Colour intensity shows how much time was logged that day. A dot indicates the day has at least one entry with notes.
  • Click any day to see exactly what was done — entries are read-only here (Log tab is the place to manage them).
  • Stats at the top: this month, this week, all time, average per active day (active = day with any logged work).

Saving your work

  • Everything saves automatically to this browser's local storage.
  • Export downloads a JSON backup. Two reasons to use it: (1) archive completed tasks from the Plan tab to the Log, (2) backup in case you switch device or reset the browser.
  • Import loads a JSON file, replacing current state. Useful when working across devices or restoring after a reset.
  • If I send you an updated version of this file: always Export before loading the new version on Netlify. Your data should carry over automatically, but Export is your safety net.

Tips

  • Don't put sensitive council data here — it's a working tool, not a secure store. Notes about people, premises, or specific records belong elsewhere.
  • Be consistent with one timer style per work block — don't run a task timer AND a session timer for the same time, or you'll double-count when you save both.
  • Re-read your Log every couple of weeks. You'll spot patterns in what's taking time vs what's actually moving the project.
